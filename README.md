# WebPart Dashboard - Техническое описание проекта

## 📋 Общая информация

**Название проекта:** WebPart Dashboard  
**Тип приложения:** ASP.NET Core MVC  
**Версия .NET:** 8.0  
**Архитектура:** Монолитное приложение с сервис-ориентированным подходом  
**Основная концепция:** Система веб-частей (web parts) для построения динамических дашбордов

---

## 🎯 Цель проекта

Разработка гибкой системы управления дашбордом, позволяющей пользователям:
- Динамически добавлять различные типы веб-частей на страницу
- Перемещать и изменять размер веб-частей с помощью drag-and-drop
- Просматривать данные в разных форматах (таблицы, графики, информеры)
- Настраивать внешний вид и содержание панели управления

## 🏗️ Архитектура проекта

### 1. Слои приложения

```
┌─────────────────────────────────────────────┐
│           Presentation Layer                │
│  (Views, CSS, JavaScript, Bootstrap)        │
├─────────────────────────────────────────────┤
│           Controllers Layer                 │
│  (DashboardController, HomeController)      │
├─────────────────────────────────────────────┤
│           Business Logic Layer              │

└─────────────────────────────────────────────┘
```

### 2. Основные компоненты

#### **Модели (Models)**
- `WebPart` - сущность веб-части
- `WebPartType` - enum типов веб-частей
- `DashboardViewModel` - модель представления дашборда

#### **Сервисы (Services)**
- `IWebPartService` - интерфейс сервиса управления веб-частями
- `WebPartService` - реализация сервиса с in-memory хранилищем

#### **Контроллеры (Controllers)**
- `DashboardController` - основной контроллер для работы с дашбордом
- `HomeController` - стандартный контроллер для начальной страницы

## 🎨 Функциональные возможности

### 1. Управление веб-частями

#### **Добавление веб-частей**
- Таблица данных (DataTable) - отображение структурированных данных
- График (Chart) - визуализация данных с помощью Chart.js
- Информер (Informer) - отображение уведомлений и системной информации

#### **Настройка веб-частей**
- Редактирование заголовка
- Изменение размера (ширина/высота)
- Перемещение по дашборду
- Удаление с дашборда
- Обновление данных в реальном времени

### 2. Drag-and-Drop функциональность

- **GridStack** библиотека для организации сетки
- Перетаскивание веб-частей в любую позицию
- Автоматическое перестроение сетки
- Сохранение позиций на сервере

### 3. Типы веб-частей

#### **Таблица данных (DataTable)**
```javascript
{
    columns: ["ID", "Название", "Статус", "Дата", "Ответственный"],
    rows: [
        ["1", "Разработка дашборда", "В работе", "2024-01-10", "Иванов И."],
        ["2", "Тестирование модуля", "Завершен", "2024-01-15", "Петров П."]
    ]
}
```

#### **График (Chart)**
```javascript
{
    type: "bar", // bar, line, pie
    labels: ["Янв", "Фев", "Мар", "Апр", "Май", "Июн"],
    data: [45, 62, 78, 85, 92, 88],
    backgroundColor: "rgba(54, 162, 235, 0.5)",
    borderColor: "rgba(54, 162, 235, 1)"
}
```

#### **Информер (Informer)**
```javascript
{
    message: "Добро пожаловать в систему!",
    type: "info", // info, warning, error
    lastUpdate: "2024-01-20T10:30:00",
    details: ["У вас 3 новых уведомления", "2 задачи требуют внимания"]
}
```

### 4. Взаимодействие с сервером

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/Dashboard/Index` | Отображение главной страницы |
| POST | `/Dashboard/AddWebPart` | Добавление новой веб-части |
| POST | `/Dashboard/RemoveWebPart` | Удаление веб-части |
| POST | `/Dashboard/UpdateWebPartPosition` | Обновление позиции |
| POST | `/Dashboard/UpdateWebPartSize` | Обновление размера |
| POST | `/Dashboard/UpdateWebPartTitle` | Обновление заголовка |
| GET | `/Dashboard/GetWebPartData/{id}` | Получение данных веб-части |

## 🔄 Бизнес-логика

### **Управление состоянием**
- Данные хранятся в памяти (In-memory) с использованием `Dictionary<int, List<WebPart>>`
- Поддержка нескольких пользователей через идентификатор
- Автоматическая генерация ID для новых веб-частей

### **Жизненный цикл веб-части**
1. **Создание** - добавление через UI, инициализация с дефолтными данными
2. **Настройка** - изменение заголовка, позиции, размера
3. **Использование** - отображение данных, обновление
4. **Удаление** - удаление с дашборда и из хранилища

### **Обработка данных**
- JSON формат для хранения данных веб-частей
- Десериализация при загрузке
- Поддержка различных структур данных для разных типов
